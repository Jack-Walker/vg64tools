nOVLins

by spinout
What it is
nOVLins is a small tool for inserting Nintendo overlay files to Zelda 64 / Animal Forest. At current, however, it only works with OoT 1.0 (Decompressed) and OoT MQ Debug, but support for other versions can be easily added by editing the source.

Usage:
python nOVLins.py [options] [OVL file] [ROM file]
Options
Note: all given integer values are read as hex when they have a '0x' prefix before them, otherwise they are read as decimal
Option Descripion
-a
 Specifies actor number. Helpful when actor number is not first halfword in data section of overlay. Usage:
-a ACTORNUMBER
-d
 Specifies the offset of the data section relative to the start of the overlay file. Usage of this is generally not needed, but if my program cannot find it, the n64 surely cannot! Usage:
-d OFFSET
-e
 Specifies endianess, either little or big. Default is big. Usage:
-e big OR -e little
-o
 Specifies physical offset of overlay file within ROM, which by default is that of the previous actor that had the same number. Usage:
-o PHYSICALOFF
-v
 Specifies version. As stated above, support is limited to OoT 1.0 Decompressed and OoT MQ Debug. Usage:
-v 1.0 OR -v debug
-va
 Specifies virtual offset of overlay file, which by default is that of the previous actor that had the same number. Usage:
-va VIRTUALOFF
